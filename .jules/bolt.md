## 2024-06-25 - Optimize Supabase Analytics Query Payload
**Learning:** Fetching full tables for statistical aggregation (e.g. `supabase.from('table').select('*')`) is a significant performance bottleneck, especially on frequently queried or potentially large endpoints like `/stats`. The codebase incorrectly fetched all columns of entire tables including large notes or descriptions just to count or tally specific fields. Additionally, `length` on data was used instead of an explicit `.select('*', { count: 'exact', head: true })`, resulting in massive redundant network transfer payloads for unneeded row bodies.
**Action:** Always constrain `.select()` clauses strictly to the required fields for aggregation metrics. When only a count is needed from a table (e.g. `totalRestaurants`), always use `{ count: 'exact', head: true }` so no data payload is retrieved, keeping network latency minimal.
